Intro to Programming Notes

1.1 Basics of the Web

World Wide Web

The WWW is a collection of html (hypertext markup language) documents.
Browsers -- the Internet -- Servers
Browsers request information of the Internet via servers using http (main protocol of the web).

HTML

Html is made up of: text content, markup, references and links.

Markup tells the computer how to display text. It is made up of tags and the whole phrase is called an element.

Attributes are references to other files - web pages, images, videos, etc.Tags can have multiple attributes.

Container elements are used to attach style for display: Span (inline) or div (block).

1.2 Creating a Structured Document

Think of your website as a house. Html provides the structure, css provides the style, and javascript provides the interactive elements.

HTML

Html is language with syntax and rules. Tags are the basic "word" of the language.The browser turns tags into elements that form a tree. An element exists between and open and close tag. Everything in between is called the content. Content can be empty, text or another element. Each tag has a type that will determine what kind of element is created -- paragraph, image, document division...Each tag can also have attributes with values.

DOM

Document Object Model (tree-like structure). It is the standard convention for representing and interacting with elements in html.

Boxes

Html pages are made up of boxes. Boxify a design -- even other shapes live in a box on the page. Boxes are created using div tags. Use class attributes to label them. Each can have multiple attributes.

1.3 Creating a Structured Document

CSS - Cascading Style Sheet

Use CSS to avoid repetition in coding. This is important for keeping code clean (not replicating bugs throughout), making changes faster, and to making the site easier and faster to load. It can be added in the html header element (only good for small projects); linked to in the html header element (larger projects); or inline with hmtl (bad practice, not recommended).

Cascading

There is a hierarchy of styles that can override each other similar to laws - fed, state, local. In CSS, lower level styles take precendence over higher level.

Inheritance

Inheritance allows for both faster coding and rendering. Ancestor-dependent relationship. Properties are applied not just to a specific element, but also its decendents.

Selector

A selector defines what elements a style will apply to. You can select by class (attribute : value). It's not necessary to memorize all the CSS selectors/properties, it's easier to reference them as needed.
